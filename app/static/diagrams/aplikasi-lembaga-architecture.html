7. Catatan desain dan titik perhatian
Sinkronisasi nilai turunan

Quota, invoice, dashboard, dan payroll bergantung pada data canonical: Enrollment, AttendanceSession, StudentPaymentLine, dan TutorPayoutLine. Saat payment/enrollment berubah, nilai turunan perlu dihitung ulang atau ditarik langsung dari sumber canonical.

Boundary yang jelas

Route menangani HTTP dan rendering; service menampung aturan bisnis; model menyimpan state. Pola ini sudah tampak kuat, tetapi file route besar seperti master.py dan payroll.py menjadi kandidat pemecahan jika fitur makin bertambah.

Operasional container

Perubahan kode pada deployment Docker perlu rebuild/restart container web. Verifikasi live sebaiknya mengecek route terproteksi dengan sesi login, bukan hanya status proses.

Integrasi WhatsApp

WhatsApp menjadi ingestion channel tambahan, bukan pengganti data utama. Hasil matching perlu tetap masuk ke model aplikasi agar laporan dan payroll memakai sumber yang sama.

Dibuat dari scan repo aktual: route blueprint, model SQLAlchemy, service layer, template Jinja, requirements, dan docker-compose.
