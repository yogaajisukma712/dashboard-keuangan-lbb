DOKUMEN KONTEKS PROYEK
Dashboard Keuangan LBB Super Smart
Target Teknologi: Flask + PostgreSQL

==================================================
1. TUJUAN DOKUMEN
==================================================

Dokumen ini menjadi context utama untuk perancangan aplikasi dashboard keuangan LBB Super Smart.

Tujuan utama sistem:
1. Memindahkan sistem pembukuan yang saat ini berjalan di spreadsheet ke aplikasi berbasis Flask dan PostgreSQL.
2. Menjaga agar seluruh angka yang ada di laporan tetap saling terhubung dan konsisten.
3. Mengubah spreadsheet yang saat ini sangat rumit menjadi sistem dengan sumber data tunggal yang lebih stabil, mudah diaudit, dan mudah dikembangkan.
4. Menjadikan seluruh ringkasan dashboard sebagai hasil perhitungan otomatis dari transaksi dasar, bukan dari input manual atau rumus silang antar sheet.

Prinsip utama:
- Spreadsheet saat ini sebenarnya sudah membentuk satu sistem pembukuan terpadu.
- Semua gambar/laporan saling terkait dan merujuk ke satu logika keuangan yang sama.
- Sistem baru harus meniru logika bisnis spreadsheet yang ada terlebih dahulu, baru setelah itu boleh disempurnakan.

5. RUMUS LOGIKA YANG BERHASIL DITANGKAP
==================================================

5.1. Total Pemasukan Bulanan
Pemasukan Siswa + Pemasukan Lain-Lain

Contoh dari data:
- Pemasukan siswa = Rp13.700.000
- Pemasukan lain-lain = Rp25.000

Maka:
Total Pemasukan Bulanan = Rp13.725.000

5.2. Margin Bulan Ini
Nominal pemasukan siswa - Hutang gaji dari pembayaran siswa

Contoh:
- Nominal siswa = Rp13.700.000
- Hutang gaji = Rp9.610.000

Maka:
Margin Bulan Ini = Rp4.090.000

5.3. Estimasi Profit Bulan Ini
Margin Bulan Ini + Pemasukan Lain-Lain - Pengeluaran Bulan Ini

Contoh:
- Margin = Rp4.090.000
- Pemasukan lain-lain = Rp25.000
- Pengeluaran = Rp1.489.536

Maka:
Estimasi Profit Bulan Ini = Rp2.625.464

5.4. Saldo Bulan Ini
Saldo bulan kemarin + total pemasukan bulanan - pengeluaran bulan ini

Contoh:
- Saldo bulan kemarin = Rp10.665.281
- Total pemasukan bulanan = Rp13.725.000
- Pengeluaran bulan ini = Rp1.489.536

Maka:
Saldo Bulan Ini = Rp22.900.745

5.5. Estimasi Sisa Saldo
Saldo Bulan Ini - Estimasi Gaji Tutor Bulan Ini

Contoh:
- Saldo bulan ini = Rp22.900.745
- Estimasi gaji tutor = Rp9.045.000

Maka:
Estimasi Sisa Saldo = Rp13.855.745

5.6. Grand Hutang Gaji
Hutang gaji lama + hutang gaji baru - pembayaran tutor

Dari pola angka yang terbaca:
- Hutang gaji bulan kemarin = Rp3.380.000
- Hutang gaji dari pembayaran siswa bulan ini = Rp9.610.000

Maka kemungkinan:
Grand Hutang Gaji = Rp12.990.000
(asumsi belum dikurangi payout tutor pada potongan data yang terlihat)

5.7. Grand Profit
Saldo kas / total saldo - grand hutang gaji

Contoh:
- Grand total saldo = Rp22.900.745
- Grand hutang gaji = Rp12.990.000

Maka:
Grand Profit = Rp9.910.745

Makna grand profit di sistem sekarang:
Saldo yang benar-benar menjadi milik lembaga setelah dikurangi kewajiban ke tutor.

==================================================
6. TEMUAN PENTING: ADA 2 JALUR PERHITUNGAN TUTOR
==================================================

Sistem saat ini memuat dua jalur hitung terkait tutor:

6.1. Hutang gaji dari pembayaran siswa
Saat siswa bayar:
- kas masuk
- sebagian nominal dialokasikan menjadi hutang gaji tutor
- sisanya menjadi margin lembaga

6.2. Estimasi gaji dari presensi tutor
Saat tutor mengajar:
- sesi tercatat di presensi
- nominal tutor terakumulasi
- muncul estimasi gaji tutor bulanan

Ini menghasilkan dua angka yang berbeda:
- Hutang gaji dari pembayaran siswa
- Estimasi gaji dari presensi

Makna:
Sistem baru harus memisahkan dengan jelas:
1. Tutor payable from collection
2. Tutor accrual from attendance
3. Tutor payout (yang benar-benar dibayarkan)

Jika tidak dipisah, dashboard akan rancu.

11. KPI DAN FORMULA DASHBOARD YANG DIREKOMENDASIKAN
==================================================

11.1. Pemasukan Siswa Bulan Ini
SUM(student_payment_lines.nominal_amount)

11.2. Pemasukan Lain-Lain Bulan Ini
SUM(other_incomes.amount)

11.3. Total Pemasukan Bulanan
Pemasukan siswa + pemasukan lain-lain

11.4. Hutang Gaji dari Pembayaran Bulan Ini
SUM(student_payment_lines.tutor_payable_amount)

11.5. Margin Bulan Ini
SUM(student_payment_lines.margin_amount)

11.6. Pengeluaran Bulan Ini
SUM(expenses.amount)

11.7. Estimasi Gaji Tutor Bulan Ini
SUM(attendance_sessions.tutor_fee_amount)
dengan status = attended

11.8. Estimasi Profit Bulan Ini
Margin bulan ini + pemasukan lain-lain - pengeluaran bulan ini

11.9. Grand Hutang Gaji
Opening tutor payable + hutang gaji bulan ini - payout tutor bulan ini

11.10. Saldo Bulan Ini
Versi 1:
Opening cash balance + total pemasukan - pengeluaran

Versi 2:
Opening cash balance + total pemasukan - pengeluaran - payout tutor

Catatan:
Perlu diputuskan versi saldo mana yang akan dipakai sebagai KPI utama.

11.11. Grand Profit
Saldo - grand hutang gaji

11.12. Estimasi Sisa Saldo
Saldo bulan ini - estimasi gaji tutor bulan ini

16. KESIMPULAN FINAL
==================================================

Sistem spreadsheet saat ini pada dasarnya adalah satu ekosistem pembukuan terpadu dengan komponen:
1. data operasional les
2. data pemasukan siswa
3. data pemasukan lain-lain
4. data pengeluaran
5. data presensi tutor
6. data rekap gaji tutor
7. data dashboard summary

Formula intinya adalah:
- nominal - hutang gaji = margin
- margin + pemasukan lain - pengeluaran = profit bulanan
- saldo awal + pemasukan - pengeluaran = saldo
- hutang lama + hutang baru - payout tutor = grand hutang gaji
- saldo - grand hutang gaji = grand profit

Arah sistem baru:
- satu database transaksi
- satu sumber kebenaran untuk semua angka
- semua summary dihitung otomatis
- semua angka bisa ditelusuri ke transaksi asal
- dashboard menjadi hasil akhir, bukan tempat input manual

Dokumen ini harus dijadikan context utama saat membangun aplikasi Flask dan PostgreSQL untuk Dashboard Keuangan LBB Super Smart.
